01

Variables y tipos

Las variables son cajitas con nombre donde guardamos información. En JS moderno usamos let (puede cambiar) y const (no puede cambiar). var existe pero es antiguo y tiene comportamientos raros, así que lo evitamos. Los tipos de datos principales son: número, texto (string), booleano, null, undefined, objeto y array.
variables_tipos.js
let   nombre   = "Ada"          // string
const edad     = 25             // number
let   activo   = true           // boolean
let   vacio    = null           // null
let   sinDef                    // undefined

console.log(typeof nombre)      // "string"
console.log(typeof edad)        // "number"
console.log(typeof activo)      // "boolean"
console.log(typeof sinDef)      // "undefined"
typeof nos dice el tipo de cualquier valor. Fíjate que typeof null devuelve "object" — es un bug histórico de JS.
02

Operadores

Los operadores nos permiten hacer cálculos y comparaciones. Los aritméticos (+, -, *, /, %) operan con números. Usa siempre === para comparar (compara valor Y tipo). Los lógicos (&&, ||, !) combinan condiciones.
operadores.js
// Aritméticos
10 + 3   // 13
10 % 3   // 1  (resto)
2 ** 8   // 256 (potencia)

// Comparación
"5" ==  5   // true  ← peligroso
"5" === 5   // false ← correcto

// Lógicos
true && false   // false
true || false   // true
!true           // false

// Ternario
const msg = edad >= 18 ? "mayor" : "menor"
El operador ternario es un if/else en una línea. El % (módulo) es perfecto para saber si un número es par (n % 2 === 0).

03

Condicionales

Los condicionales permiten tomar decisiones: si pasa esto, haz aquello. La estructura básica es if / else if / else. Para comparar un valor contra muchas opciones, switch es más limpio.
condicionales.js
const nota = 7.5

if (nota >= 9)      console.log("Sobresaliente")
else if (nota >= 7) console.log("Notable")
else if (nota >= 5) console.log("Aprobado")
else                console.log("Suspenso")

switch (dia) {
  case "lunes":   console.log("Inicio de semana"); break
  case "viernes": console.log("¡Por fin viernes!"); break
  default:        console.log("Un día normal")
}
JS evalúa los if de arriba a abajo. En el switch, el break es imprescindible. El default equivale al else.

04

Bucles

Los bucles repiten código mientras se cumpla una condición. for es ideal cuando sabes cuántas veces repetir. while repite mientras algo sea verdad. for...of recorre arrays elemento a elemento.
bucles.js
for (let i = 0; i < 5; i++) console.log(i)

let contador = 10
while (contador > 0) { console.log(contador); contador -= 2 }

const frutas = ["🍎", "🍌", "🍊"]
for (const fruta of frutas) console.log(fruta)

for (let n = 0; n < 10; n++) {
    if (n === 3) continue
    if (n === 7) break
    console.log(n)
}
break sale del bucle, continue salta a la siguiente iteración.

05

Arrays

Un array es una lista ordenada de elementos con índice desde 0. Métodos clave: push añade, pop elimina el último, filter filtra, map transforma, find busca el primero que cumpla la condición.
arrays.js
const nums = [3, 1, 4, 1, 5, 9, 2]

nums.push(6)              // añade al final
nums.pop()               // elimina el último
nums.splice(2, 1)        // elimina desde índice 2

const dobles  = nums.map(n => n * 2)
const mayores = nums.filter(n => n > 3)
const primero = nums.find(n => n > 5)

nums.includes(9)                      // true
[...nums].sort((a,b) => a - b)        // orden ascendente
map y filter devuelven nuevos arrays sin modificar el original. En sort numérico siempre pasa (a,b) => a-b.
06

Funciones

Una función es un bloque de código reutilizable. Le puedes pasar parámetros y puede retornar un resultado. Formas: declaración clásica, expresión y arrow function.
funciones.js
function saludar(nombre) {
    return `Hola, ${nombre}!`
}

const doble    = (n) => n * 2
const sumar    = (a, b) => a + b
const presentar = (nombre, rol = "alumno") => `${nombre} es ${rol}`

presentar("Ada")              // "Ada es alumno"
presentar("Tim", "profesor") // "Tim es profesor"
Les arrow functions són la forma preferida avui. Els paràmetres per defecte (amb =) eviten haver de comprovar si l'argument va arribar.
07

DOM

El DOM es la representación del HTML como árbol de objetos que JS puede leer y modificar: seleccionar elementos, cambiar contenido, estilos, añadir o borrar nodos.
dom.js
const caja = document.getElementById('miCaja')
const btn  = document.querySelector('button')

caja.textContent = "Hola mundo"
caja.style.color = "red"
caja.classList.toggle('activo')

const p = document.createElement('p')
p.textContent = "Nuevo párrafo"
document.body.appendChild(p)
p.remove()
textContent és més segur que innerHTML (evita XSS). classList.toggle afegeix o treu classes CSS automàticament.

Eventos

Los eventos son acciones del usuario (clic, tecla, scroll…) a las que podemos reaccionar. La forma recomendada es addEventListener.
eventos.js
btn.addEventListener('click', (e) => {
    console.log(e.target, e.type)
})

document.addEventListener('keydown', (e) => {
    console.log(e.key)
})

input.addEventListener('input', (e) => {
    console.log(e.target.value)
})

form.addEventListener('submit', (e) => {
    e.preventDefault()
})
e.preventDefault() evita que el formulario recargue la página. event.target apunta al elemento que disparó el evento.

Math

El objeto Math tiene constantes y métodos matemáticos listos para usar, sin necesidad de instanciarlo.
math.js
Math.floor(4.9)   // 4
Math.ceil(4.1)    // 5
Math.round(4.5)  // 5
Math.abs(-7)      // 7
Math.sqrt(25)     // 5
Math.max(3, 9, 1)  // 9
Math.min(3, 9, 1)  // 1

// Entero aleatorio entre min y max
const aleatorio = (min, max) =>
    Math.floor(Math.random() * (max - min + 1)) + min

aleatorio(1, 6)   // simula un dado
Math.random() retorna un decimal entre 0 i 1. La fórmula floor(random() * (max-min+1)) + min és el patró estàndard per enters aleatoris.

Strings

Los strings tienen métodos para buscar, cortar, transformar y combinar texto. Los template literals permiten insertar variables con ${variable}.
strings.js
const texto = "  Hola Mundo  "

texto.trim()                       // "Hola Mundo"
texto.toUpperCase()               // "  HOLA MUNDO  "
texto.includes("Mundo")           // true
texto.slice(2, 6)                  // "Hola"
texto.replace("Mundo", "JS")      // "  Hola JS  "
texto.split(" ")                   // array de palabras
String(5).padStart(2, "0")        // "05"

const saludo = `Hola, ${nombre}!`  // template literal
Els mètodes de string no modifiquen l'original, retornen un de nou. És molt comú encadenar-los: texto.trim().toLowerCase().

Fechas

El objeto Date maneja fechas y horas. Ojo: los meses van de 0 (enero) a 11 (diciembre).
fechas.js
const ahora = new Date()

ahora.getFullYear()   // 2025
ahora.getMonth()      // 0-11 (enero=0)
ahora.getDate()       // día (1-31)
ahora.getDay()        // día semana (0=domingo)

const navidad = new Date(2025, 11, 25)

// Diferencia en días
const diff = navidad - new Date()
const dias = Math.ceil(diff / (1000 * 60 * 60 * 24))

ahora.toLocaleDateString('es-ES')  // "12/05/2025"
Per calcular diferències, JS treballa en mil·lisegons. Divideix per 1000*60*60*24 per convertir a dies.

Formularios

Los formularios recogen datos del usuario. Con JS leemos valores con .value, validamos con .trim() y procesamos sin recargar la página.
formularios.js
const nombre = document.getElementById('nombre').value.trim()
const opcion  = document.getElementById('select').value
const marcado = document.getElementById('check').checked
const radio   = document.querySelector('input[name="turno"]:checked')

if (!nombre) { alert("Nombre obligatorio"); return }
if (!radio)  { alert("Selecciona un turno"); return }
.trim() evita que alguien envíe solo espacios. .checked retorna true/false per checkboxes i radios.

Errores típicos

Errores que todo el mundo comete al empezar. Conocerlos te ahorra mucho tiempo. try/catch captura errores en tiempo de ejecución sin que el programa explote.
errores_tipicos.js
if (x = 5) { }    // ❌ asigna
if (x === 5) { }  // ✅ compara

"5" == false   // ❌ true  (trampa)
"5" === false  // ✅ false

new Date(2025, 12, 25) // ❌ mes 12 no existe
new Date(2025, 11, 25) // ✅ diciembre

try {
    JSON.parse(textoInvalido)
} catch (error) {
    console.log("Error:", error.message)
} finally {
    console.log("Siempre se ejecuta")
}
El finally s'executa sempre, hi hagi error o no. Útil per tancar recursos.
